# ACP2E-4626 : chargement double des fichiers JavaScript Storefront provoquant des chargements en double et un comportement instable

Le correctif ACP2E-4626 corrige le problème où les fichiers JavaScript de storefront sont demandés et exécutés deux fois, provoquant des chargements en double intermittents et un comportement instable.Après le correctif, les dépendances de JavaScript ne sont chargées qu’une seule fois, ce qui élimine les demandes en double tout en préservant les fonctionnalités de mixin existantes. Ce correctif est disponible lorsque la version 1.1.80 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4626. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7 - 2.4.8-p5

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Problème en raison duquel certains fichiers de JavaScript Storefront étaient demandés et exécutés deux fois, ce qui provoquait des chargements en double intermittents et un comportement instable. Après le correctif, les dépendances se chargent une seule fois, ce qui élimine les demandes en double tout en préservant les fonctionnalités de mixin existantes. »

<u>Conditions préalables</u> :

1. MSI est installé.

<u>Procédure à suivre </u> :

1. Ouvrez n’importe quelle page storefront dans le navigateur.
1. Ouvrez l’**[!UICONTROL DevTools]** du navigateur et sélectionnez l’onglet **[!UICONTROL Console]** .
1. Exécutez la commande suivante.

   ```
   require.config({ deps: ['mage/common'] });
   ```

<u>Résultats attendus</u> :

Aucun chargement en double ne se produit. L’appel renvoie la fonction requise et aucune nouvelle requête de script n’est générée.

<u>Résultats réels</u> :

Les nouvelles requêtes apparaissent dans l’onglet **[!UICONTROL Network]** (par exemple, mage/common.js, requirejs/domReady.js, jquery.js).
