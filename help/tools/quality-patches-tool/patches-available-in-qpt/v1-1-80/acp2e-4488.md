# ACP2E-4488 : enregistrement/modification lente du produit par l’administration pour les jeux d’attributs volumineux

Le correctif ACP2E-4488 corrige le problème en raison duquel l’enregistrement ou la modification des produits dans Admin est lent pour les produits dotés de jeux d’attributs volumineux. Ce correctif est disponible lorsque la version 1.1.80 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4488. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

L’enregistrement ou la modification de produits avec des jeux d’attributs volumineux est lent dans l’administration.

<u>Procédure à suivre </u> :

1. Créez au moins 500 attributs de produit et affectez-les à un jeu d’attributs.
1. Accédez au panneau **Administration**.
1. Créez un produit ou modifiez un produit existant à l’aide du jeu d’attributs comportant de nombreux attributs.
1. Cliquez sur **Enregistrer**.

<u>Résultats attendus</u> :

1. L’opération d’enregistrement ou de modification du produit s’exécute correctement sans erreur de navigateur.
1. La page Admin reste réactive pendant le processus d’enregistrement.

<u>Résultats réels</u> :

1. La page d’administration ne répond plus lors de l’enregistrement ou de la modification du produit.
1. Un avertissement de navigateur *La page ne répond pas* s’affiche.

## Application du correctif

Pour appliquer des correctifs individuels, utilisez les liens suivants en fonction de votre méthode de déploiement :

* Adobe Commerce ou Magento Open Source On-premise : [[!DNL Quality Patches Tool] > Utilisation](/help/tools/quality-patches-tool/usage.md) dans le guide de [!DNL Quality Patches Tool].
* Adobe Commerce sur les infrastructures cloud : [Mises à niveau et correctifs > Appliquer des correctifs](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/upgrade/apply-patches.html) dans le guide Commerce sur les infrastructures cloud .
