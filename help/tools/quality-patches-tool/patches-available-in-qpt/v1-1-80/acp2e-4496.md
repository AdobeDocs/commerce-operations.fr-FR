# ACP2E-4496 : le traitement cron Analytics dégrade les performances pendant l’exécution

Le correctif ACP2E-4496 corrige le problème en raison duquel la tâche cron Analytics entraîne une dégradation des performances pendant l’exécution, ce qui se traduit par une amélioration des performances globales du système. Ce correctif est disponible lorsque la version 1.1.80 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4496. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

La tâche cron Analytics entraîne une dégradation des performances lors de l’exécution et augmente la charge de la base de données.

<u>Procédure à suivre </u> :

1. Activez l’analyse MBI.
1. Générer un grand nombre d&#39;entrées dans la table `sales_order_item`.
1. Exécutez la commande cron `analytics_collect_data` et mesurez le temps nécessaire à l’exécution.

<u>Résultats attendus</u> :

L’analyse de la liste d’attente doit prendre un temps raisonnable.

<u>Résultats réels</u> :

L’exécution de l’analyse par cron prend beaucoup de temps et place une charge élevée sur la base de données.
