# ACP2E-4533 : le chargement des images d’espace réservé échoue sur le storefront lorsque l’URL inclut le code du magasin

Le correctif ACP2E-4533 corrige le problème en raison duquel les images d’espace réservé ne se chargent pas sur le storefront lorsqu’un code de magasin est inclus dans l’URL. Ce correctif est disponible lorsque la version 1.1.80 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4533. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p8

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7 - 2.4.7-p10

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les images d’espace réservé ne se chargent pas sur le storefront lorsque l’URL inclut le code du magasin.


<u>Procédure à suivre </u> :

1. Créez un site web, un magasin et une vue de magasin supplémentaires.
1. Pour configurer des images d’espace réservé dans la portée par défaut, dans la barre latérale d’administration, accédez à **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]**.
1. Dans le panneau de gauche, développez **[!UICONTROL Catalog]** et choisissez **[!UICONTROL Catalog]**.
1. Développez la section **[!UICONTROL Product Image Placeholders]** pour configurer les images d’espace réservé pour la nouvelle portée du site web.
1. Créez un produit avec des images et affectez-le aux deux sites web.
1. Supprimez les images de produit chargées du dossier `pub/media/catalog/product`.
1. Ouvrez le produit sur le storefront pour les deux sites web.

<u>Résultats attendus</u> :

Le site web autre que celui par défaut doit afficher les images d’espace réservé spécifiques au site web lorsque les images réelles ne sont pas disponibles.

<u>Résultats réels</u> :

Le site web autre que celui par défaut affiche l’image d’espace réservé par défaut, quelles que soient les images d’espace réservé chargées pour celle-ci.
