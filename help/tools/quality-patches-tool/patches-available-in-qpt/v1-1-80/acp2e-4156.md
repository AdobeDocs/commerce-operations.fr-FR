# ACP2E-4156 : la validation de l’adresse d’expédition de l’API [!DNL REST] ignore la configuration des attributs d’administration

Le correctif ACP2E-4156 corrige le problème en raison duquel la validation de l’adresse d’expédition dans l’API [!DNL REST] ne respecte pas la configuration d’attribut définie dans Admin. Ce correctif est disponible lorsque la version 1.1.80 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4156. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p4

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Dans l’API [!DNL REST], la validation des adresses d’expédition ne respecte pas les paramètres d’attribut d’adresse du client configurés dans Admin.

<u>Procédure à suivre </u> :

1. Accédez à **[!UICONTROL Stores]** > **[!UICONTROL Attributes]** > **[!UICONTROL Customer Address]** dans le panneau d’administration.
1. Ouvrez l’attribut avec le code d’attribut *prénom*.
1. Sous **[!UICONTROL Attribute Properties]**, définissez :
   1. **[!UICONTROL Input Validation]** : longueur uniquement.
   1. **[!UICONTROL Minimum Text Length]**: *1*.
   1. **[!UICONTROL Maximum Text Length]**: *20*.
1. Accédez à la vitrine et ajoutez le produit test au panier.
1. Envoyez une demande d’API [!DNL REST] à `/rest/default/V1/guest-carts/cartId/shipping-information` pour ajouter ou mettre à jour des informations d’expédition avec une valeur *prénom* de plus de 20 caractères.

<u>Résultats attendus</u> :

Une erreur est renvoyée indiquant que la valeur *prénom* dépasse la limite de longueur configurée.

<u>Résultats réels</u> :

Les informations d’expédition du panier ont été mises à jour avec la valeur *prénom* envoyée dans la requête API [!DNL REST].
