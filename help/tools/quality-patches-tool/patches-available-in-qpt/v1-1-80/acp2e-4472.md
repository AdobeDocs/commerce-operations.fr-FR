# ACP2E-4472 : devis nul créé à l’aide du flux Se connecter en tant que client

Le correctif ACP2E-4472 corrige le problème où les enregistrements de guillemets nuls sont créés dans la table des guillemets pendant le flux de **[!UICONTROL Login as Customer]**. Ce correctif est disponible lorsque la version 1.1.80 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4472. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p8

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Un enregistrement de devis nul est créé dans la table des devis pendant le flux de [!UICONTROL Login as Customer].

<u>Procédure à suivre </u> :

1. Créez deux clients.
1. Accédez à **[!UICONTROL Admin]** > **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**.
1. Dans la grille des clients, sélectionnez un client et cliquez sur **[!UICONTROL Login as Customer]**.
1. Après la redirection vers le storefront, vérifiez les données dans la table de base de données `quote`.
1. Sans fermer la session storefront en cours, revenez à la grille des clients **[!UICONTROL Admin]**, sélectionnez l’autre client, puis cliquez sur **[!UICONTROL Login as Customer]**.

<u>Résultats attendus</u> :

Seul un devis associé au client connecté est créé dans la table de base de données `quote`.

<u>Résultats réels</u> :

Avec le devis du client, un enregistrement de devis nul est créé dans la table de base de données `quote`.
