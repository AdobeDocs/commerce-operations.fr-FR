# ACP2E-4552 : la réponse GraphQL ne renvoie pas le statut de la société

Le correctif ACP2E-4552 corrige le problème en raison duquel le statut de la société n’est pas renvoyé dans la réponse GraphQL. Ce correctif est disponible lorsque la version 1.1.80 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4552. Notez que ce problème doit être résolu dans la version 1.5.2-p3 d’Adobe Commerce B2B.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) B2B 1.5.2-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) B2B >=1.5.0 &lt;1.5.3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

La réponse GraphQL ne renvoie pas le statut de la société.

<u>Conditions préalables</u> :

[!DNL B2B] est installé.

<u>Procédure à suivre </u> :

1. Dans Admin, accédez à **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]**, puis définissez **[!UICONTROL Enable Company]** sur `Yes`.
1. Connectez-vous à Commerce Admin en tant qu’administrateur.
1. Créez trois sociétés avec les statuts suivants : Actif, Rejeté et Approbation en attente.
1. Dans Admin, accédez à **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**. Ouvrez un compte client, puis, dans la liste **[!UICONTROL Associate to Company]**, affectez le client à chacune des sociétés créées.
1. Envoyez la requête GraphQL suivante :

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies {
        items {
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>Résultats attendus</u> :

La réponse de GraphQL renvoie les valeurs de statut correctes (par exemple Actif, En attente d’approbation, Rejeté et Bloqué) pour toutes les sociétés associées au client. La réponse peut également être filtrée ou triée par statut d’entreprise.

<u>Résultats réels</u> :

La réponse GraphQL n’inclut pas les valeurs de statut de société pour les sociétés associées au client.
