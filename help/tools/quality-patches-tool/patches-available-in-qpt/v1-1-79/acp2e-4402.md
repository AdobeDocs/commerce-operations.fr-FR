# ACP2E-4402 : l’indexeur incrémentiel de la règle cible ne sélectionne pas les produits désactivés

Le correctif ACP2E-4402 corrige le problème où les produits créés comme désactivés n’étaient pas ajoutés aux résultats des règles Target associées après avoir été activés. Ce correctif est disponible lorsque la version 1.1.79 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4402. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.5-p13

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.5 - 2.4.5-p16

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Problème en raison duquel les produits créés comme désactivés n’étaient pas ajoutés aux résultats des règles Target associées après activation.

<u>Procédure à suivre </u> :

1. Créez un **[!UICONTROL Related Product Rule]** simple.
1. Créez un produit désactivé pour correspondre à la condition de la règle (par exemple, **[!UICONTROL Product Type]** est **[!UICONTROL Simple Product]**). Le problème n’est pas spécifique à une condition particulière.
1. Exécutez cron pour exécuter la liste d’attente de l’indexeur.
1. Activez le produit ultérieurement.
1. Exécutez cron pour exécuter la liste d’attente de l’indexeur.
1. Vérifiez la section Storefront de la **[!UICONTROL Related Products]** ou recherchez l’ID de produit nouvellement créé dans la table `magento_targetrule_product`.

Si le problème n’est pas reproductible avec les étapes ci-dessus, essayez cet autre chemin :
1. Installez une nouvelle instance.
1. Créez une catégorie.
1. Créez un **[!UICONTROL Related Product Rule]**.
1. Exécutez cron.
1. Créez un produit désactivé.
1. Exécutez cron.

<u>Résultats attendus</u> :

Les règles de Target sont mises à jour avec les produits qui ont été initialement créés comme désactivés puis activés. Le nouveau produit activé s’affiche dans la section Produits associés et dans `magento_targetrule_product`.

<u>Résultats réels</u> :

L’indexeur incrémentiel `targetrule_product_rule` ignore le produit lors de sa création, car il est désactivé. L’activation ultérieure du produit n’ajoute pas de liste d’attente pour `targetrule_product_rule`, de sorte que le produit n’apparaît jamais dans la section Produits associés. Seule une réindexation complète la récupère.
