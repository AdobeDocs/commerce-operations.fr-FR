# ACP2E-4505 : dépendance de catégorie circulaire lors de l’enregistrement d’une catégorie avec des données obsolètes dans un onglet de navigateur en double

Le correctif ACP2E-4505 corrige le problème en raison duquel il était possible d’enregistrer une catégorie avec des données obsolètes à partir d’un onglet de navigateur en double, créant ainsi une dépendance circulaire. Ce correctif est disponible lorsque la version 1.1.79 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4505. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3, 2.4.7-p8

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Il a été possible d’enregistrer une catégorie avec des données obsolètes à partir d’un onglet de navigateur en double, créant ainsi une dépendance circulaire.

<u>Procédure à suivre </u> :

1. Créez une structure de catégories : A > B > C.
1. Dans le panneau d’administration, accédez à **[!UICONTROL Catalog]** > **[!UICONTROL Categories]**.
1. Dans l’arborescence des catégories, sélectionnez la catégorie C pour ouvrir son formulaire d’édition dans l’onglet 1.
1. Dupliquez l’onglet du navigateur pour ouvrir l’onglet 2 avec le même formulaire de modification de catégorie C.
1. Dans L’Onglet 1 :
   1. Faites glisser la catégorie C sous A pour que C devienne un enfant de A.
   1. Faites glisser la catégorie B sous C pour que B devienne un enfant de C.
   1. Attendez la fin des opérations de glisser-déposer et la mise à jour de l’arborescence des catégories.
1. Dans l’onglet 2 (l’ancienne hiérarchie est toujours affichée) :
   1. Apportez des modifications, par exemple, modifiez la description.
   1. Cliquez sur **[!UICONTROL Save]**.

<u>Résultats attendus</u> :

Le système doit valider et empêcher l’enregistrement des relations parent-enfant circulaires.

<u>Résultats réels</u> :

Une dépendance parent-enfant circulaire est créée entre les catégories (la catégorie B a le parent C et la catégorie C a le parent B), ce qui entraîne une récursivité infinie dans le parcours transversal de la hiérarchie des catégories.
