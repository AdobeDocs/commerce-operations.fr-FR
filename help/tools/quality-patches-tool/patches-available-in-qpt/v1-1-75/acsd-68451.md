# ACSD-68451 : l’administration de l’entreprise fournit des liens vers des entreprises non liées sur plusieurs sites web

Le correctif ACSD-68451 corrige le problème en raison duquel un administrateur d’entreprise se connecte à un site web, crée une entreprise non liée sur un autre site web et est incorrectement associé à cette entreprise non liée. Ce correctif est disponible lorsque la version 1.1.75 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-68451. Veuillez noter que ce problème devrait être résolu dans Adobe Commerce [!DNL B2B] 1.5.3.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) - [!DNL B2B] 1.5.2-p2

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) - [!DNL B2B] 1.5.2-p1 - 1.5.2-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Dans une configuration multisite, un administrateur d’entreprise se connecte à un site web et crée une entreprise. Ensuite, dans le même navigateur, l’administrateur ouvre un autre site web et crée une autre société. La plateforme associe incorrectement le second administrateur de la société à la première société, même si les sociétés ne sont pas liées.

<u>Conditions préalables</u> :

Installez [!DNL B2B] et Inventaire.

<u>Procédure à suivre </u> :

1. Créez un site web, un magasin et une vue de magasin supplémentaires.
1. Activez *[!UICONTROL Company]* sous **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]**.
1. Accédez au site web par défaut sur le storefront.
1. Créez une nouvelle entreprise sur le storefront et connectez-vous en tant qu’administrateur d’entreprise.
1. Au cours de la même session de navigateur, accédez au site web secondaire.
1. Créez une nouvelle entreprise sur le storefront pour le site web secondaire.
1. Connectez-vous au panneau d’administration et vérifiez la grille du client.

<u>Résultats attendus</u> :

Le nouvel administrateur d’entreprise reste affecté uniquement à la société nouvellement créée, même lorsqu’un autre administrateur d’entreprise reste connecté sur un autre site web dans le même navigateur.

<u>Résultats réels</u> :

Le nouvel administrateur d’entreprise est également affecté à l’autre entreprise en tant qu’**utilisateur de l’entreprise**, en plus de l’entreprise prévue.
