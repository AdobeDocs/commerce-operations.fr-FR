# ACSD-68793 : les produits valides sont incorrectement rejetés lors de l’affectation du catalogue partagé

Le correctif ACSD-68793 corrige le problème où les produits valides sont incorrectement rejetés lors de l’affectation du catalogue partagé. Ce correctif est disponible lorsque la version 1.1.75 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-68793. Veuillez noter que ce problème devrait être résolu dans Adobe Commerce [!DNL B2B] 1.5.3.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) [!DNL B2B] 1.5.2-p1

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) [!DNL B2B] 1.5.1 - 1.5.2-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les produits valides sont incorrectement rejetés lors de l’affectation du catalogue partagé via l’API [!DNL REST] et le système renvoie une erreur indiquant que les SKU existants n’existent pas.

<u>Procédure à suivre </u> :

1. Créer plusieurs produits.
1. Dans la barre latérale **[!UICONTROL Admin]**, accédez à **[!UICONTROL Catalog]** > **[!UICONTROL Shared Catalog]**.
1. Cliquez sur **[!UICONTROL Add Shared Catalog]** et créez un catalogue partagé.
1. Affectez tous les produits au catalogue partagé que vous avez créé.
1. Envoyez une requête d’API [!DNL REST] à `/rest/V1/sharedCatalog/3/assignProducts` avec la liste des SKU des produits existants pour les affecter au catalogue partagé.

<u>Résultats attendus</u> :

Une réponse réussie est renvoyée. Les erreurs ne doivent être renvoyées que pour les produits qui n’existent pas.

<u>Résultats réels</u> :

Le système renvoie l&#39;erreur *Le produit demandé n&#39;existe pas : %sku.* pour un SKU existant.
