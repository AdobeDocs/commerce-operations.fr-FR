# ACP2E-4732 : l&#39;indexation partielle s&#39;arrête pour les clients qui ont de nombreuses mises à jour lorsque la colonne version_id de la table changelog atteint sa valeur maximale

Le correctif ACP2E-4732 corrige le problème d’arrêt de l’indexation partielle pour les clients avec un grand nombre de mises à jour lorsque la colonne version_id de la table changelog atteint sa valeur maximale. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4732. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p8

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4 - 2.4.7-p9

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

L’indexation partielle s’arrête pour les clientes et clients qui ont un grand nombre de mises à jour une fois que la colonne `version_id` dans la table de `changelog` a atteint sa valeur maximale.

<u>Conditions préalables</u> :

1. L’instance s’exécute en mode de production.
1. Les indexeurs sont définis en mode *Mise à jour selon le calendrier*.

<u>Procédure à suivre </u> :

1. Créer un produit et le mettre à jour 4 294 967 295 fois (valeur entière non signée maximale dans MySQL).
1. Mettez à jour le produit une fois de plus.

<u>Résultats attendus</u> :

1. Toutes les mises à jour sont correctement synchronisées en mode *Mise à jour selon le calendrier*.

<u>Résultats réels</u> :

1. Les produits ne sont pas affichés sur le Storefront.
1. Le tableau `catalog_product_index_price` ne contient pas d&#39;entrées de prix pour ces produits.
