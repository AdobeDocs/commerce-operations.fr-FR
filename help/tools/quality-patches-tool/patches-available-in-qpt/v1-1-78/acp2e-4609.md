# ACP2E-4609 : la page **[!UICONTROL My Quotes]** n’affiche aucun guillemet lorsque certains guillemets contiennent des produits supprimés

Le correctif ACP2E-4609 corrige le problème en raison duquel la page **[!UICONTROL My Quotes]** n’affiche aucun guillemet lorsque certains guillemets contiennent des produits supprimés. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4609. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p8

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

La page **[!UICONTROL My Quotes]** n’affiche aucun guillemet lorsque certains guillemets contiennent des produits supprimés.

<u>Conditions préalables</u> :

Les fonctionnalités Société et Devis B2B sont activées.

<u>Procédure à suivre </u> :

1. Créez une entreprise.
1. Créer plusieurs produits simples.
1. Créez une offre groupée avec au moins deux options, puis sélectionnez **[!UICONTROL User Defined]**. Attribuez les produits simples à différentes options de lot.
1. Connectez-vous au storefront en tant qu’administrateur de la société.
1. Ajoutez deux fois le produit groupé au panier à l’aide des mêmes options, mais avec des quantités différentes.
1. Accédez au panier et créez un devis.
1. Supprimez les produits simples associés.
1. Pour reproduire le problème, la citation doit être marquée comme `trigger_recollect: 1`.

   Exécutez la commande SQL suivante sur la base de données Adobe Commerce :

   ```
   UPDATE quote SET trigger_recollect = 1 WHERE entity_id = 1;
   ```

1. Accédez à **[!UICONTROL My Account]** > **[!UICONTROL My Quotes]**.

<u>Résultats attendus</u> :

**[!UICONTROL My Quotes]** grille doit s’afficher correctement.

<u>Résultats réels</u> :

1. **[!UICONTROL My Quotes]** grille n&#39;est pas renseignée.
1. Réponse HTTP (`negotiable_quote/ui/render/`) :

   ```
   {"error":"Requested quote is not found. Row ID: quoteId = 1","errorcode":"0"}
   ```

1. Erreur dans le `exception.log` :

   ```
   report.CRITICAL: Magento\Framework\Exception\NoSuchEntityException: Requested quote is not found. Row ID: quoteId = 1 in app/code/Magento/NegotiableQuote/Model/NegotiableQuoteManagement.php:673
   ```
