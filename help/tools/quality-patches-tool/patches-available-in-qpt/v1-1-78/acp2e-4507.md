# ACP2E-4507 : [!UICONTROL Password Options] configuration ne s’applique pas aux demandes de réinitialisation de mot de passe client effectuées par le biais de mutations de GraphQL

Le correctif ACP2E-4507 corrige le problème où la configuration **[!UICONTROL Password Options]** n’est pas appliquée aux demandes de réinitialisation de mot de passe client effectuées par le biais de mutations de GraphQL. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4507. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

La configuration **[!UICONTROL Password Options]** n’est pas appliquée aux demandes de réinitialisation de mot de passe client effectuées par le biais de mutations de GraphQL.

<u>Procédure à suivre </u> :

1. Vérifiez et confirmez les options de mot de passe du client dans .
   **[!UICONTROL Store]** > **[!UICONTROL Configurations]** > **[!UICONTROL Customers]** > **[!UICONTROL Customer Configuration]** > **[!UICONTROL Password Options]**.
1. Envoyez un e-mail de réinitialisation de mot de passe à l’aide de la mutation `requestPasswordResetEmail` GraphQL.
1. Réinitialisez le mot de passe à l’aide de la mutation `resetPassword` GraphQL à l’aide du jeton de l’e-mail envoyé.
1. Répétez les mêmes étapes pour réinitialiser le mot de passe plusieurs fois.

<u>Résultats attendus</u> :

Les réinitialisations de mot de passe client déclenchées via GraphQL se comportent de la même manière que les réinitialisations de mot de passe web et respectent les paramètres de configuration [!UICONTROL Admin].

<u>Résultats réels</u> :

Les demandes de réinitialisation de mot de passe effectuées via GraphQL peuvent être déclenchées un nombre illimité de fois et n’appliquent pas les limites définies dans la configuration du mot de passe du client [!UICONTROL Admin].
