# ACP2E-4456 : l’annulation d’une commande avec une mutation GraphQL ne déplace pas les commandes entièrement payées avec des cartes-cadeaux vers le statut Fermé .

Le correctif ACP2E-4456 corrige le problème en raison duquel l’annulation de commande GraphQL ne transfère pas les commandes payées par carte cadeau au statut [!UICONTROL Closed]. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4456. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Un problème se produit lorsque l’annulation d’une commande par le biais d’une mutation GraphQL ne fait pas passer les commandes payées par carte cadeau au statut [!UICONTROL Closed].

<u>Procédure à suivre </u> :

1. Créez une carte cadeau de n’importe quel type.
1. Créez un produit simple.
1. Activez **[!UICONTROL Free Shipping]**.
1. Activez l’annulation de commande GraphQL en accédant à **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL Sales]** > **[!UICONTROL Order cancellation]**, puis définissez **[!UICONTROL Order cancellation through GraphQL]** sur *Oui*.
1. Se connecter au compte storefront.
1. Achetez une carte cadeau, générez une facture pour la commande et notez le numéro de la carte cadeau.
1. Ajoutez le produit simple au panier.
1. Sélectionnez **[!UICONTROL Free Shipping]** lors du passage en caisse.
1. Effectuez le paiement à l&#39;aide de la carte cadeau.
1. Facture la commande.
1. Annuler la commande en utilisant la mutation GraphQL suivante :

   ```
   mutation {
    cancelOrder(input: { order_id: "Nw==", reason: "Other" }) {
           error
           order {
               carrier
               created_at
               email
               gift_receipt_included
               grand_total
               id
               increment_id
               number
               order_date
               shipping_method
               status
           }
       }
   }
   ```

<u>Résultats attendus</u> :

Le statut de la commande passe à Annulé.

<u>Résultats réels</u> :

La commande n’est pas annulée.
