# ACP2E-4416 : les points de récompense client ne s’initialisent pas lors de leur création dans l’administrateur

Le correctif ACP2E-4416 corrige le problème où les points de récompense client ne sont pas initialisés lors de leur création dans l’administration. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4416. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les points de récompense client ne sont pas initialisés lors de leur création dans l’administrateur.

<u>Procédure à suivre </u> :

1. Créez un client à l’aide de l’API Admin ou REST.
1. Envoyez une requête GraphQL pour récupérer le solde des points de récompense du client.
1. Examinez la réponse.

<u>Résultats attendus</u> :

Le système renvoie le solde des points de récompense du client.

<u>Résultats réels</u> :

Le système renvoie une erreur :

```
{"errors":[{"message":"Internal server error","locations":[{"line":5,"column":9}],"path":["customer","reward_points","balance","points"]}],"data":{"customer":{"reward_points":{"balance":null}}}}
```

Pour appliquer des correctifs individuels, utilisez les liens suivants en fonction de votre méthode de déploiement :
