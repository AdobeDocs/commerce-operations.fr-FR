# ACP2E-4522 : des erreurs de clé en double intermittentes se produisent sur la table `quote_coupons` lorsque plusieurs demandes d’enregistrement de fusion de panier ou de devis s’exécutent en même temps

Le correctif ACP2E-4522 corrige le problème en raison duquel une erreur intermittente de clé dupliquée se produit sur la table `quote_coupons` lorsque plusieurs demandes d’enregistrement de fusion de panier ou de devis s’exécutent en même temps. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4522. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p7

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Une erreur intermittente de clé en double se produit sur la table `quote_coupons` lorsque plusieurs demandes d’enregistrement de fusion de panier ou de devis s’exécutent simultanément.

<u>Procédure à suivre </u> :

1. Activez le nombre maximal de coupons par commande = 10 sous la barre latérale d’administration dans **[!UICONTROL Stores]** > **[!UICONTROL Settings]** > **[!UICONTROL Configuration]** > **[!UICONTROL Sales]** > **[!UICONTROL Multicoupon Settings]**.
1. Créez une règle de prix de panier sous **[!UICONTROL Marketing]** > **[!UICONTROL Promotions]** > **[!UICONTROL Cart Price Rules]** > **[!UICONTROL  Add New Rule]** :
   1. **[!UICONTROL Rule Name]** : TESTCOUPON2026
   1. **[!UICONTROL Active]** : **[!UICONTROL Yes]** ; **[!UICONTROL Websites]** : **[!UICONTROL Main Website]** ; **[!UICONTROL Customer Groups]** ; Sélectionnez tous les groupes applicables.
   1. **[!UICONTROL Coupon]** : **[!UICONTROL Specific Coupon]** ; **[!UICONTROL Coupon Code]** : TESTCOUPON2026.
   1. **[!UICONTROL Uses per Customer]** : laisser vide.
   1. **[!UICONTROL Conditions]** : aucune condition ajoutée.
   1. **[!UICONTROL Actions]** > **[!UICONTROL Percent of product price discount]**; **[!UICONTROL Discount Amount]** = 10.
1. Ajoutez un produit d’une quantité d’au moins 7 dans un panier invité à l’aide de GraphQL, puis appliquez le coupon.
1. Exécutez six appels GraphQL mergeCarts parallèles simultanés :

   ```
   mutation {
     mergeCarts(
       source_cart_id: "<GUEST_CART_ID>"
       destination_cart_id: "<DESTINATION_CART_ID>"
     ) {
       id
       items { id product { sku } }
       applied_coupons { code }
     }
   }
   ```

<u>Résultats attendus</u> :

Aucune erreur n’est renvoyée par la fusion GraphQL.


<u>Résultats réels</u> :

La mutation GraphQL mergeCarts renvoie une erreur.
Les journaux indiquent que l&#39;échec de GraphQL est dû à une violation de contrainte de clé dupliquée sur la table `quote_coupons` (QUOTE_COUPONS_QUOTE_ID_COUPON_CODE) lorsque plusieurs demandes d&#39;enregistrement de fusion ou de devis s&#39;exécutent simultanément.

```
tail -n 200 var/log/exception.log var/log/debug.log | grep -E "Duplicate entry|QUOTE_COUPONS_QUOTE_ID_COUPON_CODE"
```
