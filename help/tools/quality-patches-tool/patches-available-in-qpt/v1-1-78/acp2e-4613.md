# ACP2E-4613 : un répertoire multimédia volumineux entraîne des retards dans le chargement de l’arborescence du répertoire de la galerie de médias

Le correctif ACP2E-4613 corrige le problème en raison duquel les structures de répertoires de médias volumineuses ralentissent les réponses `gettree`, ce qui entraîne des temps de chargement plus longs pour l’arborescence de répertoires de **[!UICONTROL Media Gallery]**. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4613. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les structures de répertoires de médias volumineuses entraînent des réponses `gettree` lentes, ce qui allonge les temps de chargement de l’arborescence du répertoire [!UICONTROL Media Gallery].

<u>Conditions préalables</u> :

1. Il existe un grand nombre de produits avec plusieurs images.

<u>Procédure à suivre </u> :

1. Générez des miniatures pour augmenter l’empreinte multimédia :

   ```
   bin/magento catalog:images:resize
   ```

1. Créez plusieurs sous-répertoires sous `pub/media/catalog/category` (par exemple : test, test2, test3, test4 et d’autres dossiers imbriqués si nécessaire).
1. Copiez un grand nombre de fichiers multimédias de catalogue dans les répertoires nouvellement créés afin d’augmenter considérablement la taille de l’arborescence des répertoires de média, par exemple :

   ```
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test/
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test2/
   (...)
   ```

1. Connectez-vous au panneau d’administration et accédez à **[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**.
1. Notez que la requête `admin/media_gallery/directories/gettree/?isAjax=true` prend beaucoup de temps.

<u>Résultats attendus</u> :

L’arborescence du répertoire se charge en quelques secondes.

<u>Résultats réels</u> :

Le chargement de l’arborescence de répertoires prend plusieurs minutes, selon le nombre de sous-répertoires sous `pub/media`.
