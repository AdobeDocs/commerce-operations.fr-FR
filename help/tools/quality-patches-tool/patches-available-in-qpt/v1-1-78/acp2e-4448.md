# ACP2E-4448 : les modifications de configuration effectuées lors des pannes de [!DNL Redis] ne sont pas répercutées après la récupération de l’[!DNL Redis] et les valeurs obsolètes persistent

Le correctif ACP2E-4448 corrige le problème en raison duquel le cache local et le cache Redis distant pouvaient être désynchronisés lorsque Redis n’était pas disponible, et les modifications de configuration effectuées lors des pannes de Redis ne se reflétaient pas après la récupération de Redis, ce qui entraînait la persistance des valeurs obsolètes. Le correctif garantit une synchronisation correcte sur tous les caches et conserve les dernières valeurs enregistrées même lorsque Redis est hors service. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4448. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p1

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Le cache L2, le cache local, le cache obsolète et Redis ont été désynchronisés lorsque Redis n’était pas disponible. Les modifications de configuration effectuées lors des pannes Redis n’étaient pas prises en compte après la récupération de Redis, ce qui entraîne la persistance de valeurs obsolètes.

<u>Procédure à suivre </u> :

1. Configurez le cache L2 à l’aide du serveur principal `RemoteSynchronizedCache` avec Redis comme serveur principal distant et le cache de fichiers comme serveur principal local.
1. Accédez à **Configuration** > **Général** > **Contact** > **Options d’e-mail** > **Envoyer des e-mails à**.
1. Mettez à jour la valeur **Envoyer des e-mails à** sur _100@example.com_.
1. Actualisez la page plusieurs fois pour confirmer que la valeur est mise en cache.
1. Arrêtez le service Redis pour simuler une interruption de service.
1. Mettez à jour la valeur **Envoyer des e-mails à** sur _200@example.com_.
1. Démarrez le service Redis.
1. Actualisez la page de configuration.

<u>Résultats attendus</u> :

Doit afficher la nouvelle valeur de configuration __.

<u>Résultats réels</u> :

Elle affiche l’ancienne valeur de configuration __.
