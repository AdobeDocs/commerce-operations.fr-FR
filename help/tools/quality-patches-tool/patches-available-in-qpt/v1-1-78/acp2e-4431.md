# ACP2E-4431 : les [!UICONTROL Related Products] correspondant à la cible sont supprimés pendant le processus de réindexation

Le correctif ACP2E-4431 corrige le problème où les **[!UICONTROL Related Products]** correspondant aux règles de la cible sont supprimées pendant le processus de réindexation. Ce correctif est disponible lorsque la version 1.1.78 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4431. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p4

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les **[!UICONTROL Related Products]** qui sont associés aux règles de la cible sont supprimés pendant le processus de réindexation.

<u>Procédure à suivre </u> :

1. Créez 10000 ou plusieurs produits.
1. Attribuez 10000 produits ou plus à 5 000 catégories ou plus.
1. Créez 100 règles de produit associées ou plus.
1. Configurez les conditions de règle où la **[!UICONTROL Products]** à correspondre au **[!UICONTROL Product Category]** d’utilisation est *X*.
1. Configurez les **[!UICONTROL Products]** à afficher à l’aide de **[!UICONTROL Product Category]** est **[!UICONTROL Constant Value]** *Y*.
1. Exécutez cette requête GraphQL pour récupérer les `related_products` d’un produit :

   ```graphql
   {
       products(filter: { url_key: { eq: "simple-product-491" } }) {
           items {
               sku
               related_products {
                   uid
                   sku
                   name
                   url_key
                   stock_status
                   price_range {
                       minimum_price {
                           regular_price {
                              value
                           }
                           final_price {
                               value
                           }
                       }
                   }
               }
           }
       }
   }
   ```

1. Créez une liste d’attente importante pour l’index `targetrule_product_rule`.
1. Créez une liste d’attente importante pour l’index `targetrule_rule_product`.
1. Exécutez l’indexeur de `targetrule_rule_product`.
1. Exécutez l’indexeur de `targetrule_product_rule`.

<u>Résultats attendus</u> :

La liste `related_products` est toujours remplie avec des produits qui correspondent aux conditions de la règle.

<u>Résultats réels</u> :

La liste `related_products` renvoie un tableau vide pendant l’exécution des indexeurs.
