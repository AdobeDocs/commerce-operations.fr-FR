# ACSD-69311 : calcul de taxe incorrect dans les avoirs après remboursements partiels des factures

Le correctif ACSD-69311 corrige le problème lié aux calculs de taxe incorrects dans les avoirs lors de la création d&#39;un remboursement partiel à partir d&#39;une facture, si un avoir précédent a été créé à partir de la page d&#39;affichage de la commande. Ce correctif est disponible lorsque la version 1.1.77 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-69311. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p8

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Des calculs de taxe incorrects se produisent dans les avoirs lors de la création d&#39;un remboursement partiel à partir d&#39;une facture, si un avoir précédent a été créé à partir de la page de vue de commande.

<u>Procédure à suivre </u> :

1. Configurez les prix de catalogue pour inclure les taxes.
1. Configurez tous les prix d’affichage pour inclure les taxes.
1. Créez un taux d’imposition et une règle fiscale pour *10 %* pour *Australie*.
1. Créez 2 produits avec des prix *100* et *200*.
1. Ajoutez les deux produits au panier.
1. Extrayez en utilisant la livraison gratuite et une adresse australienne.
1. Ouvrez la commande dans le [!UICONTROL Admin] .
1. Créez une facture pour la commande.
1. Créez un avoir partiel à l&#39;aide du bouton de la page Vue des commandes sélectionnant un produit.
1. Ouvrez la facture.
1. Créez un avoir en utilisant le bouton de la facture.

<u>Résultats attendus</u> :

Le montant de taxe est correctement reflété dans l&#39;avoir du remboursement.

<u>Résultats réels</u> :

Le deuxième avoir inclut le montant total de taxe dans le total du remboursement.
