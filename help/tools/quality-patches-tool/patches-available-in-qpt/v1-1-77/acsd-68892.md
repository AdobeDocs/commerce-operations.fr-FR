# ACSD-68892 : comportement de mise en cache [!DNL Fastly] incohérent pour les pages pouvant être mises en cache

Le correctif ACSD-68892 corrige le problème où les pages pouvant être mises en cache ne sont pas correctement stockées ou diffusées à partir du cache [!DNL Fastly], ce qui entraîne un comportement de mise en cache incohérent et une réduction des performances. Ce correctif est disponible lorsque la version 1.1.77 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-68892. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p2

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les pages pouvant être mises en cache ne sont pas correctement stockées ou diffusées à partir du cache [!DNL Fastly], ce qui entraîne un comportement de mise en cache incohérent et une réduction des performances.

<u>Procédure à suivre </u> :

1. Activez le cache [!DNL Fastly].
1. Créez une vue de site web/magasin/magasin.
1. Ajoutez le code du site web secondaire à `index.php` :

```
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. Vérifiez l’en-tête de contrôle du cache à l’aide de la commande curl :

```
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u>Résultats attendus</u> :

Les pages pouvant être mises en cache sont mises en cache dans [!DNL Fastly].

<u>Résultats réels</u> :

Les pages pouvant être mises en cache ne sont pas mises en cache dans [!DNL Fastly], `max-age=0` renvoyées dans l’en-tête .

```
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```
