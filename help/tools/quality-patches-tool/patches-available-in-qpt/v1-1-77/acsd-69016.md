# ACSD-69016 : Le prix spécial n&#39;est pas appliqué sur les sites web avec des fuseaux horaires différents

Le correctif ACSD-69016 corrige le problème d’échec des demandes d’API de mise à jour [!DNL REST] de mise à jour de produit s’il n’existe aucun magasin avec le code par défaut, ce qui empêche l’application de prix spéciaux pour les sites web créés dans différents fuseaux horaires. Ce correctif est disponible lorsque la version 1.1.77 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-69016. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p1

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

La mise à jour du produit [!DNL REST] les requêtes d’API échouent s’il n’existe aucun magasin avec le code par défaut, ce qui empêche l’application de prix spéciaux pour les sites web créés dans différents fuseaux horaires.

<u>Procédure à suivre </u> :

1. Dans le panneau **[!UICONTROL Admin]**, accédez à **[!UICONTROL Stores]** > **[!UICONTROL &#x200B; All Stores]** > **[!UICONTROL Default Store View]**.
1. Modifiez la valeur de code de la vue de magasin par défaut.
1. Envoyez une requête d’API [!DNL REST] pour mettre à jour un produit.

<u>Résultats attendus</u> :

La demande est terminée avec succès et le produit est mis à jour.

<u>Résultats réels</u> :

La requête échoue et le produit n’est pas mis à jour.

## Application du correctif

Pour appliquer des correctifs individuels, utilisez les liens suivants en fonction de votre méthode de déploiement :

* Adobe Commerce ou Magento Open Source On-premise : [[!DNL Quality Patches Tool] > Utilisation](/help/tools/quality-patches-tool/usage.md) dans le guide de [!DNL Quality Patches Tool].
