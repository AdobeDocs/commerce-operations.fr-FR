# ACSD-69494 : les demandes de remboursement asynchrones avec is_online ne déclenchent pas de remboursements en ligne

Le correctif ACSD-69494 corrige le problème lié aux opérations de remboursement asynchrones où les demandes de remboursement avec le paramètre `is_online` ne sont pas traitées correctement. Ce correctif est disponible lorsque la version 1.1.77 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-69494. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les demandes de remboursement asynchrones avec le paramètre `is_online` ne sont pas traitées correctement.

<u>Procédure à suivre </u> :

1. Configurez PayPal.
1. Configurez [!DNL RabbitMQ].
1. Créez un produit.
1. Paiement avec PayPal.
1. Créez une facture.
1. Créez un avoir à l’aide de l’API [!DNL REST] asynchrone.
1. Exécutez la file d’attente asynchrone.
1. Connectez-vous à la passerelle de paiement et vérifiez le statut de la transaction.

<u>Résultats attendus</u> :

La transaction s&#39;affiche comme **Remboursée** lorsqu&#39;un remboursement en ligne est effectué.

<u>Résultats réels</u> :

La transaction apparaît comme **Terminée**.
