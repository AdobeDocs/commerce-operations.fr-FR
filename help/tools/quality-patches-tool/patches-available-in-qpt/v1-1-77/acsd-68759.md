# ACSD-68759 : la création du compte client échoue avec le paramètre régional Arabe (Arabie saoudite) lorsque le [!UICONTROL Date of Birth] est visible sur le storefront

Le correctif ACSD-68759 corrige le problème en raison duquel le chargement de la page de **[!UICONTROL Create an Account]** client échoue lorsque le **[!UICONTROL Locale]** est défini sur *Arabe (Arabie saoudite)* et que l’attribut **[!UICONTROL Date of Birth]** s’affiche sur le storefront. Ce correctif est disponible lorsque la version 1.1.77 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-68759. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p5

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4-p2 - 2.4.4-p16, 2.4.5-p1 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Le chargement de la page **[!UICONTROL Create an Account]** échoue lorsque la **[!UICONTROL Locale]** est définie sur l’arabe (Arabie saoudite) et que l’attribut **[!UICONTROL Date of Birth]** est visible sur le storefront, ce qui entraîne une `TypeError`.

<u>Procédure à suivre </u> :

1. Connectez-vous à l’administrateur.
2. Accédez à **[!UICONTROL Stores]** > *[!UICONTROL Attributes]* > **[!UICONTROL Customer]**.
3. Modifiez le code d’attribut **[!UICONTROL dob]**.
   * Définissez **[!UICONTROL Show on Storefront]** sur *Oui*.
4. Cliquez sur **[!UICONTROL Save Attribute]**.
5. Accédez à **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL Locale Options]**.
   * Définissez **[!UICONTROL Locale]** sur *Arabe (Arabie saoudite)*.
6. Allez à la vitrine.
7. Ouvrez la page **[!UICONTROL Create an Account]**.

<u>Résultats attendus</u> :

La page **[!UICONTROL Create an Account]** se charge sans erreur lorsque la **[!UICONTROL Locale]** est définie sur *Arabe (Arabie saoudite)* et que l’attribut **[!UICONTROL Date of Birth]** s’affiche sur le storefront.

<u>Résultats réels</u> :

* La page **[!UICONTROL Create an Account]** ne se charge pas.
* L’erreur suivante est consignée :

  ```
  [2025-10-31T02:14:35.029229+00:00] report.CRITICAL: TypeError: iterator_to_array(): Argument #1 ($iterator) must be of type Traversable|array, null given in app/code/Magento/Customer/Block/Widget/Dob.php:421
  ```
