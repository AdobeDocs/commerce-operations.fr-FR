# ACSD-67171 : la session [!DNL B2B] expirée pendant l’extraction est redirigée vers la page Accès refusé au lieu de la page de connexion

Le correctif ACSD-67171 corrige un problème en raison duquel les clients [!DNL B2B] sont redirigés vers une page *Accès refusé* lorsque la session expire ou est supprimée pendant le passage en caisse. Ce correctif est disponible lorsque la version 1.1.73 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-67171. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p1

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Le système affiche une page Accès refusé lors de l’extraction lorsque la session d’un utilisateur [!DNL B2B] expire ou est supprimée, au lieu de le rediriger vers la page de connexion du client.

<u>Procédure à suivre </u> :

1. Dans le panneau d’administration, accédez à [!UICONTROL Stores] > [!UICONTROL Configuration] > [!UICONTROL General] > [!UICONTROL [!DNL B2B] Features], définissez [!UICONTROL [!DNL B2B] Features] sur *Oui*, puis accédez à [!UICONTROL Customers] > [!UICONTROL Persistent Shopping Cart] et définissez [!UICONTROL Enable Persistence] sur *Yes*.
1. Accédez à [!UICONTROL Customers] > [!UICONTROL Companies] et sélectionnez [!UICONTROL Add New Company] pour créer une société [!DNL B2B].
1. Connectez-vous au storefront en tant qu’utilisateur ou utilisatrice de la société [!DNL B2B].
1. Ajoutez n’importe quel produit au panier.
1. Accédez à la page Passage en caisse.
1. Laissez la session expirer.
1. Rechargez la page de passage en caisse.

<u>Résultats attendus</u> :

Vous êtes redirigé vers la page de connexion du client.

<u>Résultats réels</u> :

Vous êtes redirigé vers `/company/accessdenied` et vous verrez *Vous n’êtes pas autorisé à afficher cette page. Si vous pensez qu’il s’agit d’une erreur, contactez l’administrateur ou l’administratrice de votre entreprise.*
