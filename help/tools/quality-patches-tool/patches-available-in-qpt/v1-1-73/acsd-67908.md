# ACSD-67908 : la fusion des fichiers JavaScript échoue dans les configurations multi-magasin

Le correctif ACSD-67908 corrige le problème en raison duquel les fichiers JavaScript ne fusionnent pas correctement dans les configurations multi-magasin. Ce correctif est disponible lorsque la version 1.1.73 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-67908. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p5

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

La génération des fichiers JavaScript fusionnés échoue dans les configurations multi-magasin, ce qui entraîne des fichiers corrompus et des erreurs de storefront.

<u>Procédure à suivre </u> :

1. Configurez au moins 50 sites web, boutiques et affichages de boutique.
1. Définissez le mode de production et ignorez la compilation :

   ```
   bin/magento deploy:mode:set production --skip-compilation
   ```

1. Déployez le code généré :

   ```
   bin/magento setup:di:compile
   ```

1. Déployez du contenu statique à l’aide d’une stratégie compacte :

   ```
   bin/magento setup:static-content:deploy --strategy compact --jobs 10
   ```

1. Supprimez le contenu du dossier du cache fusionné :

   ```
   rm -rf pub/static/_cache/merged/*
   ```

1. Effacer le cache :

   ```
   bin/magento cache:clean
   ```

1. Ouvrez simultanément toutes les vues de magasin dans des onglets ou des fenêtres de navigateur distincts.


<u>Résultats attendus</u> :

Tous les fichiers JavaScript sont fusionnés avec succès et le storefront fonctionne sans erreurs.

<u>Résultats réels</u> :

Certains fichiers JavaScript fusionnés sont corrompus et ne commencent pas par la déclaration correcte du module RequireJS.
