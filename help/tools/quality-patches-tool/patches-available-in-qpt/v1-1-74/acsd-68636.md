# ACSD-68636 : nom du propriétaire du magasin manquant dans les en-têtes des e-mails de carte cadeau lorsque la facture est créée à partir d’un autre magasin

Le correctif ACSD-68636 corrige un problème en raison duquel, après la création d’une facture à partir d’un magasin différent, le modèle d’e-mail de carte cadeau ne reflétait pas le nom du propriétaire à partir des paramètres de configuration d’administration dans les en-têtes d’e-mail. Une fois ce correctif appliqué, les en-têtes des e-mails incluent désormais les informations d’e-mail du propriétaire du magasin approprié. Ce correctif est disponible lorsque la version 1.1.74 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-68636. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p2

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Lorsqu’un e-mail de carte cadeau a été envoyé après la création d’une facture à partir d’un autre magasin, les en-têtes d’e-mail ne reflétaient pas le nom du propriétaire dans les paramètres de configuration d’administration.

<u>Procédure à suivre </u> :

1. Créez un site web secondaire, un magasin ou une vue de magasin.
1. Modifiez les **[!UICONTROL Sender Name]** et **[!UICONTROL Sender Email]** sous **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL Store Email Addresses]** > **[!UICONTROL General Contact]** dans le site web secondaire ou la portée du magasin.
1. Créez un produit Carte cadeau (virtuel) et affectez-le au deuxième site web.
1. Ajoutez la carte-cadeau au panier et passez une commande sur le deuxième site Web.
1. Facturez la commande pour envoyer l’e-mail de carte cadeau.
1. Vérifiez l’expéditeur de l’e-mail de carte cadeau.

<u>Résultats attendus</u> :

Les valeurs Contact général spécifiques au magasin doivent être utilisées pour l’e-mail.

<u>Résultats réels</u> :

L’adresse e-mail d’expéditeur par défaut est utilisée.
