# ACSD-68810 : commande affectée au client créée sur un autre site web malgré la configuration du partage de compte client

Le correctif ACSD-68810 corrige le problème d’affectation d’une commande à un client créé sur un autre site web malgré la **[!UICONTROL Account Sharing Options]** dans la configuration **[!UICONTROL Customer Configuration]**. Ce correctif est disponible lorsque la version 1.1.74 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACSD-68810. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8-p3

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Une commande est affectée à un client créé sur un autre site web, malgré les **[!UICONTROL Account Sharing Options]** de la configuration **[!UICONTROL Customer Configuration]**.

<u>Procédure à suivre </u> :

1. Créez un site web secondaire, un magasin et une vue de magasin.
1. Dans la barre latérale **[!UICONTROL Admin]**, accédez à **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL CUSTOMERS]** > **[!UICONTROL Customer Configuration]** > **[!UICONTROL Account Sharing Options]** > **[!UICONTROL Share Customer Accounts]**, puis vérifiez que le paramètre **[!UICONTROL Share Customer Accounts]** est défini sur *Par site web*.
1. Créez un produit.
1. Créez une commande sur le site web principal à l’aide de GraphQL.
1. Créez un compte client à l’aide de la même adresse e-mail dans le magasin secondaire.

<u>Résultats attendus</u> :

L’association de commandes client respecte l’étendue du partage client définie dans l’administrateur.

<u>Résultats réels</u> :

La commande est associée au compte client créé sur l’autre site web.
