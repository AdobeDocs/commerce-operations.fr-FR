# ACP2E-4709 : les pages CMS ne sont pas enregistrées dans Page Builder lorsque la hiérarchie de page inclut des guillemets doubles

Le correctif ACP2E-4709 corrige le problème où les pages CMS ne s’enregistrent pas dans le générateur de page lorsque la hiérarchie de page inclut des guillemets doubles. Ce correctif est disponible lorsque la version 1.1.81 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4709. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p9

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7 - 2.4.8-p5

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

Les pages CMS ne sont pas enregistrées dans le générateur de page lorsque la hiérarchie de page comprend des guillemets doubles.

<u>Conditions préalables</u> :

Page Builder est activé.

<u>Procédure à suivre </u> :

1. Dans la barre latérale d’administration, accédez à **[!UICONTROL Stores]** > **Paramètres** > **Configuration**. Dans le panneau de gauche, sous **[!UICONTROL General]**, choisissez **Gestion de contenu**.
1. Développez **[!UICONTROL CMS Page Hierarchy]**, définissez **[!UICONTROL Enable Hierarchy Functionality]** sur **[!UICONTROL Yes]**, puis cliquez sur **[!UICONTROL Save Config]**.
1. Dans la barre latérale d’administration, accédez à **Contenu** > **[!UICONTROL Elements]** > **[!UICONTROL Pages]** et cliquez sur **[!UICONTROL Add New Page]**.
1. Saisissez un titre de page qui comprend des guillemets doubles, placez la page sous la racine du site web dans l’onglet **Hiérarchie**, puis cliquez sur **[!UICONTROL Save]**.
1. Utilisez Page Builder pour ajouter du contenu à la page, puis cliquez sur **[!UICONTROL Save]**.

<u>Résultats attendus</u> :

La nouvelle page CMS est enregistrée sans erreur.

<u>Résultats réels</u> :

- La nouvelle page CMS n’est pas enregistrée.
- La console du navigateur affiche le message d’erreur suivant : `Uncaught SyntaxError: JSON.parse: expected ',' or '}' after property value in object at line 1 column 79 of the JSON data`.
