# ACP2E-4786 : l’exportation du produit échoue lorsque le stockage distant AWS S3 est configuré

Le correctif ACP2E-4786 corrige le problème d’échec de l’exportation du produit, car le pilote de stockage utilise un chemin non valide lorsque le stockage distant AWS S3 est activé. Ce correctif est disponible lorsque la version 1.1.81 de [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) est installée. L’ID du correctif est ACP2E-4786. Notez que ce problème doit être résolu dans Adobe Commerce 2.4.9.

## Produits et versions concernés

**Le correctif est créé pour la version Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.7-p4

**Compatible avec les versions d’Adobe Commerce :**

* Adobe Commerce (toutes les méthodes de déploiement) 2.4.5 - 2.4.9

>[!NOTE]
>
>Le correctif peut s’appliquer à d’autres versions avec de nouvelles versions de [!DNL Quality Patches Tool]. Pour vérifier si le correctif est compatible avec votre version d’Adobe Commerce, mettez à jour le package `magento/quality-patches` vers la dernière version et vérifiez la compatibilité sur la page [[!DNL Quality Patches Tool] : Rechercher des correctifs](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=fr). Utilisez l’ID du correctif comme mot-clé de recherche pour localiser le correctif.

## Problème

L’exportation du produit échoue lorsque le stockage distant AWS S3 est activé en raison d’une gestion de chemin non valide dans le pilote de stockage.

<u>Procédure à suivre </u> :

1. Configurez [!DNL AWS S3] comme stockage distant :

   ```
   bin/magento setup:config:set --remote-storage-driver="aws-s3" --remote-storage-bucket="..." --remote-storage-region="..." --remote-storage-key="..." --remote-storage-secret="..." -n
   ```

1. Désactivez le stockage de la base de données pour les fichiers multimédias en exécutant la commande suivante :

   ```
   bin/magento config:set system/media_storage_configuration/media_database 0
   ```

1. Dans la barre latérale d’administration, accédez à **[!UICONTROL System]** > **[!UICONTROL Data Transfer]** > **Exporter**.
1. Par **[!UICONTROL Entity Type]**, choisissez **[!UICONTROL Products]**.
1. Cliquez sur **[!UICONTROL Continue]**.

<u>Résultats attendus</u> :

Le fichier d’exportation a été créé.

<u>Résultats réels</u> :

1. Le fichier d’exportation n’est pas créé.
1. Les journaux affichent une erreur *SignatureDoesNotMatch*.
